API Reference

REST endpoints for identity lifecycle management.

The API is organized around agents, DIDs, keys, credentials, capabilities, trust, revocations, and audit events.

Core Endpoints

POST   /v1/agents
GET    /v1/agents/{agent_id}
GET    /v1/dids/{did}
POST   /v1/agents/{agent_id}/keys
POST   /v1/agents/{agent_id}/credentials
POST   /v1/agents/{agent_id}/capabilities
GET    /v1/agents/{agent_id}/trust
POST   /v1/revocations
GET    /v1/audit-events
