AgenNext Identity Control Plane

Persistent, verifiable identity for AI agents.

Agent Identity gives every autonomous agent a cryptographic identity, DID document, verifiable credentials, capability metadata, trust score, and audit trail.

Explore API Read Docs

Who is this agent?

Resolve a stable DID and verify the keys associated with the agent.

Can it be trusted?

Evaluate credentials, issuer trust, revocation state, reputation events, and trust score.

What can it do?

Attach capability grants and constraints that downstream authorization systems can enforce.

Architecture

Identity lifecycle for autonomous systems

Register agents, generate keys, publish DID documents, issue credentials, rotate keys, revoke access, and preserve audit history.

Schema

Production-ready data model

The recommended schema supports organizations, developers, agents, keys, DID documents, credentials, grants, revocations, trust scores, reputation events, and audit events.

organizations
  └── developers
        └── agents
              ├── agent_keys
              ├── did_documents
              ├── verifiable_credentials
              ├── capability_grants
              ├── trust_scores
              ├── reputation_events
              └── audit_events
Deploy

Run locally or deploy to the cloud

Use Docker Compose for local deployment and GitHub Actions for CI/CD. The service is designed to run cleanly on Render, Railway, Fly.io, Cloud Run, ECS, or Kubernetes.

cp .env.example .env
docker compose up --build

curl http://localhost:8000/health
curl http://localhost:8000/ready
