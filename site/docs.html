Documentation

Build agent identity into your runtime.

Register agents, resolve DIDs, issue credentials, grant capabilities, revoke compromised identities, and audit every lifecycle event.

Concept

Agent DID

Every agent receives a stable decentralized identifier. The DID resolves to a DID document containing verification keys, authentication methods, assertion methods, and service endpoints.

did:agennext:agent:7be23bb1-f998-4b47-bc32-eaf1d72185dc
Concept

Agent Credential

Credentials bind an agent DID to claims such as name, organization, model provider, capabilities, issuer, expiration, and proof metadata.

{
  "type": ["VerifiableCredential", "AgentIdentityCredential"],
  "issuer": "did:agennext:issuer:identity",
  "credentialSubject": {
    "id": "did:agennext:agent:...",
    "agentName": "Research Agent",
    "trustLevel": "high"
  }
}
Lifecycle

Identity lifecycle

1. Register agent
2. Generate key
3. Publish DID document
4. Issue credential
5. Grant capabilities
6. Verify trust
7. Rotate keys
8. Revoke when needed
